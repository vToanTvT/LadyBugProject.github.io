Detailed Description

PWM driver for LLC converter and fan control.

This file implements the initialization and main process logic of the PWM Driver module. It provides core functions for PWM generation, output control, duty cycle adjustment, frequency configuration, and dead-time management. The module manages PWM outputs for LLC power converter control and cooling fan speed regulation using STM32 hardware timers with support for complementary outputs and configurable dead-time for safe half-bridge switching operations.

int pwm_disable_output_ ( S_Pwm_Config_TypeDef * ppwmConfig)

Disable PWM output.

Disables the PWM output for the specified timer channel. Deactivates the corresponding channel in CCER register, disables main output (MOE bit), and stops the timer counter.

Parameters
[in]ppwmConfigPointer to S_Pwm_Config_TypeDef structure containing timer and channel configuration.
Return values
SUCCESSPWM output disabled successfully.
ERRORInvalid parameter (NULL pointer, invalid instance, or unsupported channel).
Note
This function safely shuts down the PWM output including complementary outputs. Used for protection or controlled shutdown.

Disables the PWM output for the specified timer channel by performing:

  • Deactivates corresponding channel bits in CCER register, disabling both direct and complementary PWM outputs
  • Clears MOE (Main Output Enable) bit in BDTR register to disable all timer outputs
  • Stops timer counter by clearing CEN bit in CR1 register

This provides a safe shutdown sequence for PWM outputs, ensuring both main and complementary outputs are turned off properly to prevent shoot-through in half-bridge configurations.

Parameters
[in]ppwmConfigPointer to S_Pwm_Config_TypeDef structure containing the timer handle and channel configuration.
Return values
SUCCESSPWM output disabled successfully.
ERRORInvalid parameter (NULL pointer, invalid timer instance, or unsupported channel number).
Note
This function safely disables both the main channel and its complementary output. Used for protection shutdown or controlled power stage disable. Only channels 1, 2, and 3 are supported.

int pwm_enable_output_ ( S_Pwm_Config_TypeDef * ppwmConfig)

Enable PWM output.

Enables the PWM output for the specified timer channel. Activates the corresponding channel in CCER register, enables main output (MOE bit), and starts the timer counter. Supports channels 1, 2, 3 with complementary outputs.

Parameters
[in]ppwmConfigPointer to S_Pwm_Config_TypeDef structure containing timer and channel configuration.
Return values
SUCCESSPWM output enabled successfully.
ERRORInvalid parameter (NULL pointer, invalid instance, or unsupported channel).
Note
Only channels 1, 2, and 3 are supported. The function configures both direct and complementary outputs for half-bridge operation.

Enables the PWM output for the specified timer channel by performing:

  • Activates corresponding channel bits in CCER register for complementary PWM output (both direct and complementary channels)
  • Sets MOE (Main Output Enable) bit in BDTR register to enable timer outputs
  • Starts timer counter by setting CEN bit in CR1 register

Supports channels 1, 2, and 3 with their complementary outputs. Each channel configuration enables both the direct channel (CHx) and complementary channel (CHxN) for half-bridge operation.

Parameters
[in]ppwmConfigPointer to S_Pwm_Config_TypeDef structure containing the timer handle and channel configuration.
Return values
SUCCESSPWM output enabled successfully.
ERRORInvalid parameter (NULL pointer, invalid timer instance, or unsupported channel number).
Note
Only channels 1, 2, and 3 are supported. Channel selection is specified in the ppwmConfig->channel field. Both main and complementary outputs are enabled for half-bridge configurations.

◆ pwm_init_()

void pwm_init_ ( )

Initialize PWM module.

Initializes PWM configurations for both LLC converter and cooling fan. Configures timer handles, channels, frequency, duty cycle, and dead-time for both PWM outputs. Starts the underlying base timers and enables fan output by default.

Return values
None
Note
Must be called once during system startup before using PWM outputs. This function automatically enables the fan PWM output after initialization.

Performs complete initialization of PWM Driver module:

  • Starts hardware timer base for LLC and Fan
  • Configures timer handles and channels
  • Sets default frequency, duty cycle, and dead-time for LLC
  • Configures fan PWM with initial parameters
  • Enables fan output by default

The function initializes two PWM outputs:

  1. LLC converter PWM (TIM1, Channel 3) - with dead-time for half-bridge
  2. Cooling fan PWM (TIM2, Channel 1) - simple PWM for speed control
Return values
None
Note
Must be called once during system startup before using any PWM functions. The fan is automatically started after initialization.

int pwm_set_deadtime_ ( S_Pwm_Config_TypeDef * ppwmConfig,
uint32_t deadtime )

Set PWM dead-time.

Configures the dead-time value for complementary PWM outputs to prevent shoot-through in half-bridge configurations. Calculates the DTG bits in BDTR register based on the requested dead-time in nanoseconds.

Parameters
[in,out]ppwmConfigPointer to S_Pwm_Config_TypeDef structure.
[in]deadtimeDesired dead-time value in nanoseconds [ns].
Return values
SUCCESSDead-time set successfully.
ERRORInvalid parameter (NULL pointer or invalid instance).
Note
Dead-time is critical for safe half-bridge operation. The function applies dead-time only if the calculated value is within valid range (≤ 0xFF). Dead-time is inserted between complementary outputs.

Configures the dead-time value for complementary PWM outputs to prevent shoot-through in half-bridge configurations. Calculates the DTG (Dead-Time Generator) bits in BDTR register based on the requested dead-time in nanoseconds.

The calculation converts nanoseconds to timer clock cycles using: DTG_value = deadtime_ns * DEADTIME_FACTOR where DEADTIME_FACTOR = timer_clock / 10^9

The function only applies the dead-time if the calculated value fits within the 8-bit DTG field (≤ 0xFF). The DTG field controls the dead-time insertion between complementary outputs.

Parameters
[in,out]ppwmConfigPointer to S_Pwm_Config_TypeDef structure containing PWM configuration.
[in]deadtimeDesired dead-time value in nanoseconds [ns].
Return values
SUCCESSDead-time set successfully (or value out of range).
ERRORInvalid parameter (NULL pointer or invalid timer instance).
Note
Dead-time is critical for safe half-bridge operation to prevent both switches from conducting simultaneously. The function preserves other BDTR register bits while updating only the DTG field. Maximum configurable dead-time depends on timer clock frequency.

Set PWM duty cycle.

Updates the PWM duty cycle for the specified configuration. Calculates the compare register value based on the requested duty percentage and updates the appropriate CCR register (CCR1, CCR2, or CCR3) depending on the configured channel.

Parameters
[in,out]ppwmConfigPointer to S_Pwm_Config_TypeDef structure.
[in]dutyDesired duty cycle percentage (0-100) [%].
Return values
SUCCESSDuty cycle updated successfully.
ERRORInvalid parameter (NULL pointer or invalid instance).
Note
Duty cycle is applied immediately to the running PWM. Only channels 1, 2, and 3 are supported. The duty cycle affects both direct and complementary outputs symmetrically.

Updates the PWM duty cycle by calculating the compare register value based on the requested duty percentage and current timebase period. The calculation is: CCR = (duty * ARR) / 100.

Updates the appropriate capture/compare register (CCR1, CCR2, or CCR3) depending on the configured channel. The duty cycle change takes effect immediately on the running PWM signal.

Parameters
[in,out]ppwmConfigPointer to S_Pwm_Config_TypeDef structure containing PWM configuration.
[in]dutyDesired duty cycle percentage (0-100) [%].
Return values
SUCCESSDuty cycle set successfully.
ERRORInvalid parameter (NULL pointer or invalid timer instance).
Note
Only channels 1, 2, and 3 are supported. The duty cycle affects both the direct and complementary outputs symmetrically. Valid duty cycle range is 0-100%. The compare value is updated immediately without waiting for an update event.

int pwm_set_frequency_ ( S_Pwm_Config_TypeDef * ppwmConfig,
uint32_t freq )

Set PWM frequency.

Updates the PWM frequency by calculating and setting the ARR register value. Automatically recalculates the duty cycle to maintain the previously set percentage. Frequency change takes effect immediately.

Parameters
[in,out]ppwmConfigPointer to S_Pwm_Config_TypeDef structure.
[in]freqDesired PWM frequency in Hz.
Return values
SUCCESSFrequency updated successfully.
ERRORInvalid parameter (NULL pointer or invalid instance).
Note
Changing frequency will update ARR register and recalculate CCR to maintain the same duty cycle percentage. Ensure frequency is within valid range for the timer clock source.

Updates the PWM frequency by calculating and setting the auto-reload register (ARR) value based on the system clock and desired frequency. The calculation is: ARR = FREQ_CLK / freq.

After updating the frequency, the function automatically recalculates and updates the duty cycle to maintain the previously set duty percentage. This ensures the duty cycle ratio remains consistent when the frequency changes.

Parameters
[in,out]ppwmConfigPointer to S_Pwm_Config_TypeDef structure containing PWM configuration.
[in]freqDesired PWM frequency in Hz.
Return values
SUCCESSFrequency set successfully.
ERRORInvalid parameter (NULL pointer or invalid timer instance).
Note
Changing frequency updates the ARR register and automatically recalculates the CCR value to maintain the same duty cycle percentage. Ensure the requested frequency is within valid range for the timer clock source (calculated ARR should fit in timer resolution).
