LadyBug Firmware Documentation

1. Project Overview

The LadyBug Project is a high-performance firmware platform developed by EVSELab, specifically engineered for an 840W EV Charger system. It is optimized for the STM32G0B1KBUx microcontroller, focusing on real-time control, modularity, and industrial-grade safety.

2. System Architecture

The firmware utilizes a three-layer hierarchical design to ensure strict separation of concerns and high maintainability:

Layer Responsibility Key Components
Application High-level logic Charging State Machine, User Interface
Middleware Control & Signal Processing PFC/LLC Control Loops, Q15/F32 Math, Fault Management
Driver Hardware Abstraction HAL, Peripheral Drivers (ADC, PWM, DMA, UART, CAN)

3. Hardware-Software Interface

  • Core: 32-bit ARM Cortex-M0+ CPU, frequency up to 64 MHz
  • Operating Temperature: −40°C to 85°C / 105°C / 125°C
  • Memories:
    • Up to 512 Kbytes of Flash memory with protection and securable area
      • Two banks
      • Read-while-write support
    • 144 Kbytes of SRAM (128 Kbytes with HW parity check)
  • DMA: 12-channel DMA controller with flexible mapping
  • ADC:
    • 12-bit, 0.4 µs conversion time (up to 16 external channels)
    • Up to 16-bit resolution with hardware oversampling
    • Conversion range: 0 to 3.6 V
  • DAC: Two 12-bit DACs with low-power sample-and-hold
  • Communication Interfaces: UART, SPI, I²C, FDCAN
  • Timers (15 total):
    • Two 128 MHz capable timers (16-bit, advanced motor control)
    • One 32-bit general-purpose timer
    • Six 16-bit general-purpose timers
    • Two basic 16-bit timers
    • Two low-power 16-bit timers
    • Two watchdog timers
    • SysTick timer

4. Key Requirement

  • Control:
    • LLC resonant converter with frequency control
    • Duty control for light-load efficiency
    • Soft-start mode
  • Measurement:
    • Output voltage, battery voltage, and output current monitoring
    • AC input voltage monitoring
    • Temperature sensing (LLC, PFC)
  • Protection:
    • Over-voltage, over-current, and over-temperature protection
    • Short-circuit protection
    • Fault management with auto-recovery
  • Communication:
    • UART for debug and monitoring
    • GPIO control for status indication
