Detailed Description

Communication middleware for UART data transmission management.

This file implements the middleware logic for communication operations. It provides abstraction layer between application and driver for serial communication management. The module wraps driver layer UART functions and provides higher-level interface for data transmission, allowing application code to remain independent of specific hardware driver implementation details.

Transmit data via communication interface.

Initiates data transmission over UART by calling the driver layer transmit function. This middleware function provides abstraction for communication operations, allowing the application layer to send data without direct driver layer interaction.

The function delegates to uart_transmit_() which handles DMA-based transmission of the TXBuffer content.

Return values
None
Note
This function is a wrapper around the UART driver layer. Ensure TXBuffer is populated with data before calling. The transmission is non-blocking and completes asynchronously via DMA. Use HAL callbacks for transmission completion notification.

Transmit data via communication interface.

Initiates data transmission over UART communication channel by calling the driver layer uart_transmit_() function. This middleware function provides an abstraction layer that decouples application logic from hardware-specific driver implementation.

The function delegates the actual transmission to the UART driver which handles DMA-based non-blocking data transfer. The transmission sends the complete TXBuffer content configured at driver level.

Return values
None
Note
This is a wrapper function that abstracts driver layer functionality. Before calling this function, ensure the TXBuffer in driver layer is properly populated with the data to transmit. The transmission is asynchronous; use HAL callbacks (HAL_UART_TxCpltCallback) at driver level for completion notification.
