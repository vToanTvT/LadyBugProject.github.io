Detailed Description

UART driver for serial communication using DMA.

This file implements the initialization and main process logic of the UART Driver module. It provides core functions for DMA-based UART communication including non-blocking transmission and reception. The module manages transmit and receive buffers for efficient serial data transfer with external devices or debugging interfaces using DMA for CPU-independent data transfers.

HAL_StatusTypeDef uart_receive_ ( )

Receive data via UART using DMA.

Initiates DMA-based reception into the RXBuffer. The function is non-blocking and returns immediately after starting the DMA transfer. Received data will be stored in RXBuffer. Reception completion should be monitored via interrupt callback or status checking.

Return values
HAL_OKReception started successfully.
HAL_BUSYUART peripheral is currently busy with another operation.
HAL_ERRORHardware error occurred during reception setup.
Note
This function starts continuous reception into RXBuffer (UART_RX_BUFFER_SIZE bytes). The function returns immediately; actual reception happens asynchronously via DMA. Use HAL callbacks (e.g., HAL_UART_RxCpltCallback) to process received data.

Initiates DMA-based reception into the RXBuffer using HAL_UART_Receive_DMA function. The operation is non-blocking and returns immediately after starting the DMA transfer. The DMA controller automatically stores received data from UART RX register into RXBuffer without CPU intervention.

The function sets up continuous or fixed-length reception of UART_RX_BUFFER_SIZE bytes. Reception progress and completion should be monitored through HAL interrupt callbacks.

Return values
HAL_OKDMA reception started successfully. Data will be stored in RXBuffer as it arrives.
HAL_BUSYUART peripheral is currently busy with another reception or transmission operation.
HAL_ERRORHardware error occurred during DMA setup or UART peripheral error detected.
Note
This function starts reception into RXBuffer (UART_RX_BUFFER_SIZE bytes). The function returns immediately; actual reception happens asynchronously via DMA. Use HAL_UART_RxCpltCallback() to process received data when buffer is full, or HAL_UART_RxHalfCpltCallback() for half-transfer notification. Use HAL_UART_ErrorCallback() for errors.

Transmit data via UART using DMA.

Initiates DMA-based transmission of the entire TXBuffer content via UART peripheral. The function is non-blocking and returns immediately after starting the DMA transfer. Transmission completion should be monitored via interrupt callback or status checking.

Return values
HAL_OKTransmission started successfully.
HAL_BUSYUART peripheral is currently busy with another operation.
HAL_ERRORHardware error occurred during transmission setup.
Note
This function transmits the complete TXBuffer (UART_TX_BUFFER_SIZE bytes). Ensure TXBuffer is properly populated before calling. The function returns immediately; actual transmission happens asynchronously via DMA. Use HAL callbacks to detect completion.

Public macro definitions.

Initiates DMA-based transmission of the entire TXBuffer content via UART peripheral using HAL_UART_Transmit_DMA function. The operation is non-blocking and returns immediately after starting the DMA transfer. The DMA controller handles the actual data transfer to UART TX register without CPU intervention.

The function transmits all UART_TX_BUFFER_SIZE bytes from TXBuffer. Transmission progress and completion should be monitored through HAL interrupt callbacks (HAL_UART_TxCpltCallback).

Return values
HAL_OKDMA transmission started successfully. Data transfer is in progress.
HAL_BUSYUART peripheral is currently busy with another transmission or reception operation.
HAL_ERRORHardware error occurred during DMA setup or UART peripheral error detected.
Note
This function transmits the complete TXBuffer (UART_TX_BUFFER_SIZE bytes). Ensure TXBuffer is properly populated before calling this function. The function returns immediately; actual transmission happens asynchronously via DMA. Use HAL_UART_TxCpltCallback() to detect transmission completion or HAL_UART_ErrorCallback() for errors.
